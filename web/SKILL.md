# Piston によるコード実行と採点

## 構成
- `api/internal/piston/client.go`: Piston にコードを送って実行する。入力は {language, code, stdin}、出力は {stdout, stderr, 終了コード, 実行時間} を返す。
- `api/internal/service/judge.go`: 問題の隠しテストケースを取得し、コードを各テストの stdin で1回ずつ実行して stdout を期待値と比較する。全テスト通過なら「正解」。

## ルール
- 1実行ごとに時間・メモリ・出力サイズの上限を設定する。タイムアウトやエラーは「不正解（または実行失敗）」として扱う。
- 出力比較は末尾の空白・改行を正規化してから行う（仕様で別途定めない限り、生の完全一致は避ける）。
- 隠しテストの入力・期待出力は API レスポンスに含めない。返すのは合否と最小限の情報（例: 落ちたテスト数）だけ。
- Piston はローカルの compose ネットワーク内に置き、インターネットに公開しない。Go バックエンド経由でのみアクセスする。
